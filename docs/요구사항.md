# 요구사항 구체화하기

## 구현할 기능목록

* 버킷 생성
* 객체 업로드 및 다운로드
* 객체 버전
* 버킷 내 객체 목록 출력

---

## 데이터의 크기

* 최대 10GB
* 이를 효율적으로 저장할 수 있어야 함
* 매일 추가되는 데이터 10GB

→ 한 달 안정적 운영
→ 약 3TB 규모

---

## 내구성 · 가용성

* 99.99% 내구성 보장
* 99.99% 가용성 보장

---

≪ S3의 핵심 설계 전략 ≫

1. 각 저장 노드는 독립적이어야 한다
2. 수평 확장이 가능한 구조
3. 장애를 항상 발생한다고 가정
4. 데이터와 메타데이터를 분리
